.html files

index.html

This is the file that is immediately loaded by the browser.

In every .html file, the <head> contains all of the .js and .css files needed.

The <header>. is also common in index.html and appProposal.html.

There is a <script> section and the bottom of the file that contains the script to display all of the apps from firebase. This creates a new AppItem (in appItem.js) object and gets the element field from it.

login.html

Pretty basic page. It contains all of the .js and .css files

Contains a form that calls logIn() function in buttonFunction.js

createAccount.html

Similar to the login page, it calls createAccount() in the buttonFunctions.js

appProposal.html

This page includes a form for proposing apps. (Not implemented yet)

.js files

firebase.js

This file includes the Firebase API. DO NOT CHANGE

session.js

The idea behind this file is that it basically maintains a user's session on the website.

It contains a function that gets run as soon as the page has loaded (the $(document).ready(function(){. This calls the initializeFirebase() that Firebase gave us.

It also controls the login system. When the page loads, it checks the sessionStorage for a current user. sessionStorage is a data structure provided by the browser using a key (in this case user). If the sessionStorage returns nothing for user, then no one is logged in. It logs this to the console. Otherwise, it sets the activeUser global variable to whatever is in there.

Another thing about the sessionStorage. When a user logs in, the system stores a string of a JSON object. JSON is JavaScript Object Notation
